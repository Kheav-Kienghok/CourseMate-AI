# Security Policy

## Reporting a Vulnerability

If you discover a security issue, please report it privately.

- Preferred: GitHub Security Advisories
- Alternative: security@eduvision.live

Please include:
- A clear description of the issue
- Steps to reproduce
- Any relevant logs or screenshots
- Impact assessment (if known)

---

## Response Expectations

- We will acknowledge reports within 48 hours
- We will investigate and provide updates during the process
- Critical issues will be prioritized and fixed as soon as possible

---

## Scope

This project includes:

- Telegram bot logic
- Backend services
- Database interactions
- Docker configuration

Any security issue affecting these components is considered in scope.

---

## Responsible Disclosure

Please do not publicly disclose vulnerabilities until they are resolved.

We appreciate responsible disclosure and will work with reporters to coordinate fixes and communication.

---

## Notes

This project is a personal / early-stage application and is actively evolving.  
Security improvements are ongoing and welcomed.
